# Gray Code

## Problem

Generate a Gray code sequence of length `n`.

The sequence must contain all `2ⁿ` binary strings of length `n`, and every two consecutive strings must differ in exactly one bit.

---

## Problem Type

- Constructive Algorithm
- Recursion

---

## Core Idea

Construct the Gray code iteratively. Starting from the empty string, each iteration doubles the current sequence by prefixing `0` to the existing strings and `1` to the same strings in reverse order. Reversing the second half guarantees that consecutive strings differ by exactly one bit.

---

## Algorithm Template

```text
Read n

Initialize the sequence with an empty string

Repeat n times

    Prefix 0 to every current string

    Prefix 1 to every current string in reverse order

    Replace the current sequence

Print the final sequence
```

---

## Key Idea

Technique:
- Recursive Construction
- Constructive Algorithm

Mathematical / Algorithmic Insight:
- Each iteration doubles the number of valid Gray codes.
- Reversing the second half preserves the one-bit difference between consecutive strings.
- Every generated string is unique, and all `2ⁿ` bit strings are produced exactly once.

---

## Common Mistakes

- Appending the second half in the same order instead of reversing it.
- Forgetting to prefix `0` and `1` before combining the two halves.
- Assuming Gray codes are generated by ordinary binary counting, which does not guarantee that consecutive strings differ by exactly one bit.

---

## Time Complexity

**O(n · 2ⁿ)**

**Justification:**

A total of `2ⁿ` strings of length `n` are generated, and constructing each string requires `O(n)` time.

---

## Memory Complexity

**O(n · 2ⁿ)**

**Justification:**

All `2ⁿ` strings of length `n` are stored simultaneously in memory.
